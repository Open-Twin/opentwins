Safety & Privacy

Built to protect your accounts

🛡️
Human-like behavior
Agents engage at natural pace with configurable rate limits. Daily and weekly caps on comments, posts, reactions and connections prevent unnatural spikes that trigger platform detection.
⏰
Active hours only
Agents only run during your configured waking hours (e.g. 8am-11pm). No posting at 3am. Looks exactly like a real person's activity pattern.
🌐
Real browser, not API abuse
Agents use a real Chrome browser with your login session - the same way you'd browse manually. No scrapers, no unofficial APIs, no detectable automation headers.
🔒
100% local
Everything runs on your machine. Credentials never leave your computer. No cloud service, no third-party access to your accounts. You can inspect every line of code.
FAQ

Frequently asked questions

Will my accounts get banned? +
The risk is low but not zero - as with any automation. OpenTwins mitigates this by using a real browser (not API abuse), respecting rate limits, only engaging during natural hours and producing genuinely unique comments (not templates). You control the pace. Start conservative and increase gradually.
How much does it cost to run? +
OpenTwins itself is free and open source. But agents need Claude to think - so you need either a Claude Code subscription (~$20/month for Max) or Anthropic API credits (~$2-5/day depending on how many platforms and how active). The Usage tab in the dashboard tracks costs in real time.
Can I review what my twin posts? +
Yes. The Activity tab shows a full replay of every session - what the agent thought, what it typed and what it posted. You can also run agents manually (one click) and watch in real time before enabling the auto-scheduler.
What if something goes wrong? +
Every agent run is logged. If an agent hits an error, it stops and reports it in the dashboard. You can set banned phrases to prevent specific content. Rate limits cap daily activity. And you can pause or stop any agent instantly from the dashboard.
Do I need to be a developer? +
You need Node.js installed, but the setup is a web wizard - no config files to edit. If you can run npm install in a terminal, you're good. The dashboard handles everything else through a visual UI.
Is this against platform Terms of Service? +
Most platforms prohibit "automated" engagement in their ToS. OpenTwins uses a real browser and produces original, contextual content - not mass-generated spam. It behaves like a very consistent human user. That said, use at your own discretion and start with conservative rate limits.
